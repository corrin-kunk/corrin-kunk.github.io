HTML Semantic Tags

In HTML, Semantic Tags clearly describe what content is and how it is meant to function on a webpage. In other words, the content directly reflects the meaning of the tag itself. For example, tags like <header>, <article>, and <footer> each indicate their specific roles within a webpage. This makes it easier for both developers and browsers to understand the structure and purpose of the content.

Other benefits of using semenatic tags:

  • Improve accessibility for screen readers and assistive technologies (AT)
  • Create clearer page structure and easier navigation
  • Make websites more compatible across different browsers and devices
  • Help code stay compatible with future technologies

Non-Semantic Tags, on the other hand, do not provide any information about the content they contain. Examples of non-semantic tags like <div> <span>, are primarily used for styling and layout purposes. They require extra context, such as class names or CSS coding, to explain their purpose. It is important to use non-semantic tags sparingly and only when necessary, as they can make the code less readable and harder to maintain.
